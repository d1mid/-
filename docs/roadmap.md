# Roadmap

## Уже подготовлено

- создан каталог товаров по теме сантехники;
- заполнен набор интентов для диалогов;
- добавлен базовый модуль обработки текста;
- создан каркас проекта под ML, рекомендации и Telegram;
- зафиксировано требование держать структуру проекта простой и учебно-понятной.

## Что нужно сделать дальше по руководству

1. Привести `dialogues.txt` к тематическому датасету вопрос-ответ для сантехники.
   - использовать его для нейтральных, общеразговорных и полутематических реплик;
   - не смешивать его с основными товарными интентами.
2. Завершить модуль обработки ввода:
   - очистка;
   - коррекция опечаток;
   - лемматизация через `Natasha`;
   - извлечение сущностей;
   - сентимент;
   - классификация по темам.
3. Обучить распознавание интентов на `TfidfVectorizer + LinearSVC`.
4. Реализовать загрузку каталога и подбор товаров по категории, бюджету, назначению и признакам.
5. Реализовать рекламные сценарии минимум для 3 товаров из каталога.
6. Собрать общую функцию бота:
   - сначала intent из `intents.json`;
   - затем подбор, информация о товаре, сравнение или реклама;
   - затем поиск по `dialogues.txt` для нейтральных и околотематических реплик;
   - затем fallback.
7. Подключить Telegram.
8. Протестировать:
   - интенты;
   - предобработку;
   - подбор товаров;
   - рекламные переходы;
   - fallback и `dialogues.txt`.

## Принцип реализации

- Не усложнять структуру папок и код без необходимости.
- Оставлять только те директории и модули, которые реально используются в проекте.
- Делать решения такими, чтобы их можно было спокойно объяснить в студенческой курсовой работе.

## Цель по оценке

- Ориентир: уровень `отлично`.
- Для этого нужны:
  - ML для анализа намерений;
  - `dialogues.txt`;
  - Telegram API;
  - рекламные сценарии;
  - минимум 3 рекламируемых товара.
