# Roadmap

## Уже подготовлено

- создан каталог товаров по теме сантехники;
- заполнен набор интентов для диалогов;
- добавлен базовый модуль обработки текста;
- создан каркас проекта под ML, рекомендации и Telegram;
- зафиксировано требование держать структуру проекта простой и учебно-понятной.
- добавлен отдельный `domain detector` для различения сантехнических и свободных запросов;
- ослаблена эвристика `_looks_like_offtopic()`: конструкции вида `расскажи про ...` уходят в свободную тему только при отсутствии признаков сантехники;
- объединена логика тем: `theme_keywords.json` и определение темы текста теперь используются через общий сервис.

## Принцип реализации

- Не усложнять структуру папок и код без необходимости.
- Оставлять только те директории и модули, которые реально используются в проекте.
- Делать решения такими, чтобы их можно было спокойно объяснить в студенческой курсовой работе.

## Цель по оценке

- Ориентир: уровень `отлично`.
- Для этого нужны:
  - ML для анализа намерений;
  - `dialogues.txt`;
  - Telegram API;
  - рекламные сценарии;
  - минимум 3 рекламируемых товара.

## Предложения По Улучшению Кода

- Добавить в `ConversationState` память о последней свободной теме, чтобы короткие продолжения вроде `а еще?`, `а почему?`, `а Audi?` обрабатывались как продолжение предыдущего разговора.
- Для свободных тем использовать не только ответы из `dialogues.txt`, но и короткие шаблоны продолжения темы, чтобы диалог был связнее даже при маленьком датасете.
- Почистить и структурировать `dialogues.txt`, чтобы в нем оставались только нормальные разговорные пары без шумных, случайных и литературных реплик.
- Ослабить частоту возврата к сантехнике: предлагать переход не после каждого нетоварного ответа, а, например, раз в `2-3` свободные реплики.
- Расширить контекст товарного диалога: хранить не только `last_recommendation_ids`, но и последнюю категорию, последний товар в фокусе и последний тип товарного вопроса.
- Сделать rule-based обработку коротких ссылочных реплик сильнее: `первый`, `второй`, `этот`, `тот`, `а подешевле`, `а лучше`.
- При необходимости вынести часть эвристик из `bot.py` в отдельный модуль маршрутизации, чтобы основной pipeline стал короче и понятнее.
