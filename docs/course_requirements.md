# Памятка по руководству курсовой

Этот файл служит общей опорой для проекта и фиксирует, какие требования из руководства мы берем в работу.

## Тема проекта

- Вариант: бот для продажи сантехники.

## Что требуется по руководству

- Обработка ввода:
  - очистка текста;
  - исправление опечаток;
  - учет расстояния Левенштейна;
  - лемматизация или аккуратный стемминг;
  - классификация намерений;
  - извлечение сущностей;
  - сентимент-анализ;
  - классификация по темам.
- Генерация ответа:
  - ответы по интентам;
  - fallback-ответы;
  - сценарный переход к рекламе товаров;
  - поиск ответа по датасету `dialogues.txt`.
- Расширение возможностей:
  - ML для распознавания интентов;
  - использование `dialogues.txt`;
  - подключение Telegram.

## Что берем в проект

- Каталог сантехники в `products.json`.
- Интенты в `intents.json`.
- Датасет диалогов в `dialogues.txt`.
- Рекламные сценарии минимум для 3 товаров.
- ML-модель на `TfidfVectorizer + LinearSVC`.
- Telegram-интеграцию.

## Что не делаем

- Обработка голосовых команд.
- Озвучивание ответов голосом.
- Любые другие дополнительные возможности, не нужные для текущей курсовой.

## Дополнительные ограничения проекта

- Структура папок должна быть простой и понятной, без лишних уровней вложенности и неиспользуемых директорий.
- Все основные файлы данных должны лежать прямо в папке `data`, без дополнительных подпапок, если в них нет реальной необходимости.
- В проекте не нужно строить сложную архитектуру ради архитектуры.
- Работа студенческая, поэтому решения должны быть достаточно простыми, читаемыми и объяснимыми на защите.
- Усложнение допустимо только там, где оно действительно улучшает функциональность и соответствует требованиям руководства.

## Выбранные технологии

- Python
- `scikit-learn` для ML
- `Natasha` для русскоязычной лемматизации и базового извлечения сущностей
- расстояние Левенштейна в собственной реализации
- `python-telegram-bot` для Telegram

## Практическая трактовка требований

- `intents.json` используется для целевых сценариев бота по сантехнике: подбор товара, цена, характеристики, реклама, сравнение и другие прикладные запросы.
- `dialogues.txt` используется не для предметных сценариев каталога, а для нейтральных, общеразговорных и полутематических реплик, которые помогают поддерживать диалог и мягко возвращать пользователя к теме сантехники.
- `dialogues.txt` не должен быть случайной свалкой художественных или нерелевантных реплик.
- Лемматизация через `Natasha` используется по возможности, при отсутствии библиотеки работает fallback-нормализация.
- Сентимент-анализ и классификация тем реализуются в упрощенном словарном виде, что соответствует руководству.
- Функции обработки текста должны возвращать не только нормализованный текст, но и дополнительные признаки для последующей логики бота.
- При выборе между сложным и простым решением предпочтение отдается более простому, если оно не ухудшает нужный функционал бота.
- Общая логика ответа должна быть такой:
  - сначала попытка понять целевой интент из `intents.json`;
  - затем применение прикладной логики каталога, рекомендаций и рекламы;
  - затем поиск ответа в `dialogues.txt` для нейтральных и полутематических реплик;
  - затем fallback-ответ, если ничего подходящего не найдено.
